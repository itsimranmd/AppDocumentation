checkCredit

fun checkCredit(): Unit

Api call to get the user credit count. credits decide whether a user gets the mobile number of the profile if they click the contact button. if they have more than 0 zero credits, on contacting a profile they will get their mobile number and this will cost them one credit for that. if they have zero credits than on trying to contact a profile they will be show plans(Plans Fragment) according to their profile type.

Parameters

id - user's id as per login/sign-in credentials

is_lead - user's approval status as per login/sign-in credentials

Exceptions

StaticData.TAG_CONNECTIVITY_ERROR_MESSAGE - if no internet connection and we send user back to login page
